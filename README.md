# LootGen

LootGen — консольная утилита на Python для симуляции выпадения редких игровых предметов (лута) из сундуков и оценки эмпирических вероятностей дропа.

## Функциональность

- Симуляция заданного числа открытий сундуков.
- Набор предметов с разной редкостью и базовой вероятностью выпадения.
- Подсчет числа выпадений каждого предмета.
- Расчет эмпирической вероятности для каждого предмета на основе проведенной симуляции.
- Красивый табличный вывод результата в консоль.

## Технологии

- Python 3.11+
- rich (форматирование вывода, таблицы и цветной текст)
- flake8 (статический анализ кода)
- Git
- GitHub
- GitHub Actions (автоматический запуск линтера при push и Pull Request)

## Структура проекта

- lootgen.py — основной скрипт консольной утилиты.
- requirements.txt — список зависимостей с фиксированными версиями.
- .gitignore — файл исключений для Git.
- .flake8 — конфигурация линтера flake8.
- README.md — документация проекта.

## Установка и запуск

### 1. Клонирование репозитория

Откройте терминал и выполните:

```bash
git clone https://github.com/BorisovaRina/LootGen_exam.git
cd LootGen_exam
```

### 2. Создание и активация виртуального окружения

Создайте виртуальное окружение:

```bash
python -m venv venv
```

Активируйте его.

Linux и macOS:

```bash
source venv/bin/activate
```

Windows (PowerShell):

```bash
venv\Scripts\activate
```

### 3. Установка зависимостей одной командой

```bash
pip install -r requirements.txt
```

В результате будут установлены rich и flake8 в строго зафиксированных версиях.

### 4. Запуск утилиты

```bash
python lootgen.py
```

Утилита попросит ввести количество сундуков для симуляции. Введите целое неотрицательное число и нажмите Enter. После завершения расчета в консоли появится таблица с предметами, их редкостью, количеством выпадений и эмпирической вероятностью.

## Проверка качества кода

В проекте используется flake8. Конфигурация находится в файле .flake8.

Для локального запуска проверок выполните:

```bash
flake8 .
```

Код в основной ветке должен проходить проверку без ошибок и предупреждений.

## Автоматизация и CI

В проекте настроен базовый пайплайн GitHub Actions. Файл конфигурации находится по пути .github/workflows/ci.yml.

Пайплайн автоматически запускается:

- при push в ветку main
- при push в ветки вида feature/*
- при создании и обновлении Pull Request

Последовательность шагов пайплайна:

- клонирование репозитория
- установка Python нужной версии
- установка зависимостей из requirements.txt
- запуск flake8 для проверки кода

## Требования к окружению

- Установленный Python версии не ниже 3.11
- Возможность использования виртуального окружения (модуль venv)
- Доступ к сети для однократной установки зависимостей командой pip install -r requirements.txt

После выполнения шагов из раздела «Установка и запуск» проект LootGen должен запускаться и проверяться без дополнительных вопросов и ручных настроек.
